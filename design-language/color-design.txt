1. System Identity & Agent Directives
Theme Name: The Precision Volt (Light Mode)
Role: Light Mode Automotive SaaS
Primary Tech Stack: React, Next.js, Tailwind CSS

ü§ñ Agent Instructions (Strict Rules)
When generating UI components or modifying styles, you MUST adhere to the following directives:

Light Mode Default: Assume the application is always in light mode. Use light backgrounds with white cards and dark text.

Contrast Enforcement:
- Sidebar (Lime background): Always use text-graphite-900 for icons and labels
- White Cards: Use gray-900, gray-700, gray-600 for text hierarchy
- No pure white text on light backgrounds (except for special overlays like profile pictures)

No Pure Black: Do not use #000000. Use graphite-900 (#0F172A) or gray-900 (#111827) as the darkest shades.

Semantic Fallbacks: If a specific color is not specified for a "Success" state, default to Teal/Emerald, never Green (to avoid conflict with Brand Lime).

4. Typography Rules
Agents: strictly enforce font weights and colors.

Headings (H1-H3): text-gray-900 or text-graphite-900, font-bold/semibold, tracking-tight.
Body Text: text-gray-700 or text-gray-600.
Meta Data (Dates/Labels): text-gray-600 or text-gray-400.
Numbers (Prices/VINs): Use font-mono or tabular-nums for alignment.

7. Accessibility (A11y) Checks
Agents: Perform these checks on generated code.

Check: Is text-graphite-900 used on bg-brand (sidebar)?
If Yes: ‚úÖ Pass.
If No: ‚ùå Fail. Correct to dark text.

Check: Is the font size on inputs at least text-base (16px)?
Reason: Prevents auto-zoom on mobile devices.

Check: Do icon-only buttons have an aria-label?
Reason: Mechanics using screen readers or voice control.

Check: Is there sufficient contrast on white cards?
Reason: Ensure text-gray-700 or darker for body text on white backgrounds.
