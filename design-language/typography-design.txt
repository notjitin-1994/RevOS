System Identity & Agent DirectivesTheme Name: Precision IndustrialRole: Garage Management SaaS (Dark Mode Native)Tech Stack: React, Next.js, Tailwind CSSü§ñ Agent Instructions (Strict Rules)When generating UI components or modifying typography, you MUST adhere to the following directives:The "VIN" Rule (CRITICAL):IF the data is a Vehicle Identification Number (VIN), Part Number, SKU, License Plate, IP Address, or a columnar Price...THEN you MUST apply the font-mono class.Reason: Distinguishes 0 vs O and ensures vertical alignment.The "Label" Rule:IF the text is a metadata label (e.g., "Status", "Created At", "Customer")...THEN you MUST use: text-xs font-medium uppercase tracking-wider text-graphite-400.Reason: Creates visual hierarchy distinct from the data values.Header Distinction:NEVER use font-sans for H1, H2, or H3 elements.ALWAYS use font-display (Barlow) for headings to maintain the industrial brand identity.No Serif Fonts:NEVER introduce serif fonts (Times New Roman, Garamond, etc.). The system is strictly Sans/Mono.2. 4. Setup Configuration (For Reference)4.1 layout.tsx (Next.js Font Loader)Agents: Ensure variable names match the Tailwind config.TypeScriptimport { Barlow, Inter, JetBrains_Mono } from 'next/font/google'

const barlow = Barlow({
  subsets: ['latin'],
  weight: ['500', '600', '700'],
  variable: '--font-barlow',
})

const inter = Inter({
  subsets: ['latin'],
  variable: '--font-inter',
})

const mono = JetBrains_Mono({
  subsets: ['latin'],
  variable: '--font-mono',
})

export default function RootLayout({ children }) {
  return (
    <html lang="en" className={`${barlow.variable} ${inter.variable} ${mono.variable}`}>
      <body className="font-sans antialiased bg-graphite-900 text-white">
        {children}
      </body>
    </html>
  )
}
