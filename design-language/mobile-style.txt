1. System Identity & Mobile Philosophy
Theme Name: Precision Volt Mobile (Light Mode)
Core Concept: "The Clean Precision Interface."
Primary Constraint: The user is likely standing, has dirty hands, or is holding a tool in one hand.
Interaction Model: Thumb-driven navigation, large touch targets (>44px), and high contrast for outdoor/bay visibility.

2. ðŸ¤– Agent Instructions (Strict Rules)
Role: You are a Mobile-First UX Engineer.
Directives for Code Generation:

The "Thumb Zone" Rule:
NEVER place primary actions (Save, Start Job) at the top left of the screen.
ALWAYS place primary actions in a Fixed Bottom Bar, Floating Action Button (FAB), or the bottom 30% of the screen.

The "Touch Target" Rule:
All interactive elements (buttons, inputs, icons) MUST have a minimum height/width of 44px (or h-11/p-3 in Tailwind).
Reason: Preventing mis-taps with gloves or fingers.

Input Zoom Prevention (iOS):
IF creating an <input> or <select> for mobile...
THEN you MUST ensure the font size is text-base (16px).
Reason: Prevents the OS from auto-zooming when the input is focused.

No Hover States:
Do not rely on hover: for critical information (tooltips). Mobile users cannot hover.
Use active: states to provide immediate feedback on tap.

3. Layout & Grid Strategy (Responsive Breakpoints)

3.1 Breakpoint Philosophy
We write code Mobile-First. The base classes apply to mobile, and md: applies to tablet/desktop.

Breakpoint		Width		Usage Context
Default		< 640px		Phone (Portrait). Stacked layout, mobile bottom nav.
sm			640px+		Large Phone / Small Tablet. 2-column grids allowed.
md			768px+		Tablet (Portrait). Sidebar appears (collapsible).
lg			1024px+		Tablet (Landscape) / Laptop. Full Dashboard Grid.

3.2 The "Stack-to-Grid" Pattern
Agent Instruction: Convert complex dashboard widgets into linear stacks for mobile.

Desktop: 3 Columns (Widget A | Widget B | Widget C).
Mobile: Vertical Stack with vertical spacing space-y-4.

4. Navigation Patterns (Light Mode)

4.1 Mobile Bottom Navigation Bar
Instead of a sidebar, use a sticky bottom nav for core routes on mobile.

Height: h-16 (64px).
Background: bg-white + border-t border-gray-200.
Z-Index: z-50.
Icons: text-gray-600 (inactive), text-gray-900 (active).
Safe Area: Ensure pb-safe (Tailwind safe-area plugin) is used for iPhone Home bars.

4.2 Sidebar (Desktop Only, Brand Accent)
On desktop (md: breakpoint), use the lime sidebar:

Background: bg-brand (#CCFF00).
Icons/Text: text-graphite-900.
Border: border-r border-brand-hover.

4.3 Bottom Sheets (The "Drawer")
Do not use center-screen modals on mobile (they are hard to reach). Use Slide-up Bottom Sheets for forms or filters.

Trigger: Tap action.
Animation: Slide up from bottom (translate-y-0).
Backdrop: bg-gray-900/80 backdrop-blur-sm (for light mode contrast).
Drag Handle: A small w-12 h-1.5 bg-gray-300 rounded-full centered at the top of the sheet.

9. Accessibility (Mobile Specifics)

System Font Scaling: Ensure layouts do not break if the user has their OS font size set to "Large." Use flexible heights (h-auto or min-h-[44px]) rather than fixed heights.

Light Mode Outdoor Visibility: The white cards with gray-900 text provide excellent contrast for outdoor use. Ensure borders (border-gray-200 or border-gray-300) are sufficient to define button edges in bright sunlight.

Keypad Triggers:
For VINs: Use inputmode="text".
For Mileage/Price: Use inputmode="decimal" or numeric.
For Email: Use inputmode="email".
For URL: Use inputmode="url".
