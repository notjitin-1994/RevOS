1. System Identity & Agent Directives
Theme Name: Precision Schematic
Library Standard: Lucide React (Preferred) or Heroicons (Fallback).
Visual Style: 2px Stroke, Rounded Joints, Technical/Geometric.

ðŸ¤– Agent Instructions (Strict Rules)
When generating UI components involving icons, you MUST adhere to the following directives:

The Library Rule: Always import icons from lucide-react. Do not use FontAwesome or Material Icons (they look too generic/web-1.0).

The "Stroke" Rule: Use Outlined (Stroke) icons for all UI elements. Never use Solid/Filled icons unless it is a specific toggle state (e.g., a filled star for "Favorited").

Sizing Standards:
Button Icons: w-4 h-4 (16px) or w-5 h-5 (20px).
Navigation Icons: w-5 h-5 (20px) or w-6 h-6 (24px).
Feature/Hero Icons: w-8 h-8 (32px) or larger.

Color Inheritance: Never hardcode colors on the icon SVG itself. Always use Tailwind text classes (e.g., text-gray-700) on the parent or the icon component so it inherits currentColor.

Usage: Action buttons in toolbars and cards.

  return (
    <button className="

5. Accessibility & UX Directives

5.1 Decorative Icons
If an icon is purely visual (e.g., next to a label "Settings"), it must be hidden from screen readers to avoid clutter.

Code: <Settings2 className="w-4 h-4" aria-hidden="true" />

5.2 Standalone Icons
If an icon is the button (e.g., a trash can with no text), it MUST have a label.

JavaScript
<button aria-label="Delete Invoice">
  <Trash2 className="w-4 h-4 text-status-error" />
</button>

5.3 Click Targets
Icons are small (20px), but fingers are big. Never put an onClick directly on the icon SVG. Put it on the parent <button> or <div> and ensure that parent has padding (p-2) to create a minimum 44x44px touch target.
