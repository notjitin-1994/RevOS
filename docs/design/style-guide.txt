The Precision Volt System: Frontend Architecture
Version: 2.0 (Light Mode)
Design Identity: Industrial Precision meets Digital Velocity.
Core Philosophy: A "Light Mode" environment optimized for high-contrast legibility and clarity. The interface balances high data density (for parts/diagnostics) with large touch targets (for tablet-based workflow).

1. Foundations

1.1 Color System: "The Precision Volt"
A high-contrast light mode palette using clean white cards on light gray-blue backgrounds with vivid Lime sidebar for brand identity.

12		Status Success		Feedback			#2DD4BF (text-status-success)	Use Teal. Never Green

1.2 Typography: "Precision Industrial"
A tri-font system designed to distinguish data types instantly.

Headings (font-display): Barlow. Used for structure and branding. Always distinct from UI text.
UI Text (font-sans): Inter. Invisible, highly legible interface text.
Data (font-mono): JetBrains Mono. CRITICAL RULE: Must be used for VINs, License Plates, IP Addresses, and columnar prices to distinguish 0 vs O.

1.3 Iconography: "Schematic Stroke"
Library: Lucide React (Preferred)
Style: 2px Stroke, Rounded Joints
Behavior: Icons are text-gray-700 by default. Sidebar icons are text-graphite-900. On hover/active, they maintain dark colors on light backgrounds.

2. Space & Layout Strategy (Updated for Light Mode)

2.1 Spacing Scale (The 4px Standard)
We utilize a tight 4px base grid to maintain density without clutter.

Token		Size		Role
gap-1		4px		Tightest grouping (e.g., Tag text vs Icon).
gap-2		8px		Component internal spacing (e.g., Button icon vs Label).
gap-4		16px		Standard Component Spacing. Distance between inputs in a form.
gap-6		24px		Container padding for Cards (p-6).
gap-8		32px		Section separation (e.g., Header vs Content).

2.2 The "Bay View" Layout Grid
Unlike standard SaaS, garage apps need a Sidebar-first approach for tablet accessibility.

Sidebar (w-64): Fixed width, collapsible. Contains Core Navigation icons with bg-brand background.
Sticky Header (h-16): Contains global search (VIN lookup) and user profile. z-index: 50.
Main Canvas: Fluid width.

Dashboard Mode: 12-Column Grid (grid-cols-12). Widgets span 3, 4, or 6 columns.
Data Mode: Single column, full-width tables for inventory/parts.

2.3 Depth & Elevation (Light Mode Physics)
In light mode, we use shadows and borders to create depth on white cards.

Level 0 (Base): App Background (#ecf0f5)
Level 1 (Card): White Surface (bg-white) + Border (border-gray-200) + Shadow (shadow-card).
Level 2 (Float): Elevated with Shadow (shadow-lg or shadow-2xl).

3. UI Component Patterns (Light Mode)

3.1 Buttons & Actions

Primary (Dark Button):
bg-gray-700 text-white hover:bg-gray-600 font-semibold.

Secondary (Light Button):
border border-gray-300 text-gray-700 hover:bg-gray-100.

Sidebar Links (Brand Background):
bg-brand hover:bg-brand-hover/30 text-graphite-900.

Icon-Only Actions:
Rule: Must have aria-label
Target Size: Parent container must have p-2 to create minimum 44x44px touch target.

3.2 Data Displays (The "VIN" Pattern)
For every instance of technical data, follow the Label + Mono Value structure:

JavaScript
<div className="flex flex-col gap-1">
  {/* LABEL: Uppercase, tracking-wider, gray-600 */}
  <span className="text-xs font-medium uppercase tracking-wider text-gray-600">
    VIN / Serial
  </span>
  {/* VALUE: Font-Mono, Gray-900, Tracking-wide */}
  <span className="font-mono text-sm text-gray-900 tracking-wide">
    1HG CM8 263 3A 004352
  </span>
</div>

3.3 Status Indicators
Badge Style: inline-flex items-center px-2.5 py-0.5 rounded-full text-xs font-medium.
Background: 10% opacity of the status color (bg-status-success/10).
Border: 20% opacity border (border-status-success/20).
Icon: Precede text with a 12px status icon (<CheckCircle2 className="w-3 h-3 mr-1" />).

4. Motion & Micro-interactions
Research-based: "Industrial Smoothness".
To feel "World Class," the app uses specific timing functions. We avoid "bouncy" springs (too playful). We use "Hydraulic" Easing (fast start, smooth stop).

Global Transition: transition-all duration-200 ease-out.
Hover Effects: Cards brighten or darken borders slightly. hover:border-gray-300.
Active States: active:scale-[0.98] for tactile feedback.

5. Unified Agent Directives (For LLMs)
Role: You are a UX Engineer specialized in React, Next.js, and Tailwind.

Strict Directives:

Light Mode Default: The app is light by default (bg-[#ecf0f5], bg-white for cards).
The "VIN" Rule: If data is alphanumeric (VIN, Part #, Price), you MUST use font-mono.
The "Label" Rule: Metadata labels must be text-xs uppercase tracking-wider text-gray-600.
No Pure Black: Never use #000000. Use #111827 (gray-900) or #0F172A (graphite-900).
Icon Library: Always import from lucide-react.
Accessibility:
- Inputs must be text-base (16px) to prevent iOS zoom.
- Contrast: Ensure text-gray-700 or darker on white backgrounds.
- Headers: Always use font-display (Barlow) for H1-H3. Never font-sans.

6.2 Global CSS Reset
Add to globals.css to enforce the light mode base.

CSS
@layer base {
  body {
    @apply bg-[#ecf0f5] text-gray-900 font-sans antialiased;
  }

  h1, h2, h3 {
    @apply font-display tracking-tight;
  }

  *:focus-visible {
    @apply outline-none ring-2 ring-brand ring-offset-2 ring-offset-gray-900;
  }
}
